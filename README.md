## 🏗️ Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    CLIENT (Browser / PWA)                │
│                                                         │
│  Next.js 15 App Router  │  Zustand State  │  Socket.io  │
│  React 19 Components    │  Firebase SDK   │  Web Push   │
└───────────────────────────────┬─────────────────────────┘
                                │ HTTPS
┌───────────────────────────────▼─────────────────────────┐
│               NEXT.JS API ROUTES (Edge / Node)           │
│                                                         │
│  /api/auth      → Firebase token verify → JWT issue     │
│  /api/providers → Search, filter, geo-sort              │
│  /api/bookings  → Create, update, cancel                │
│  /api/admin     → Analytics, verify, approve            │
│  /api/upload    → Cloudinary image upload               │
└──────┬────────────────┬───────────────┬─────────────────┘
       │                │               │
┌──────▼──────┐  ┌──────▼──────┐  ┌───▼──────────────────┐
│  PostgreSQL  │  │  Cloudinary │  │  Firebase / Twilio   │
│  (Neon DB)  │  │  (CDN/imgs) │  │  Nodemailer / Push   │
│  via Prisma │  │             │  │  Web Push (VAPID)    │
└─────────────┘  └─────────────┘  └──────────────────────┘
```

## 🚀 Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL database (or [Neon](https://neon.tech) free tier)
- Firebase project (for auth)
- Cloudinary account (for image uploads)
- Gmail account with App Password (for email notifications)

### 1. Clone & Install

```bash
git clone https://github.com/Yajat-Sharma/localservices.git
cd localservices
npm install
```

### 2. Environment Variables

Create a `.env.local` file in the root:

```env
# Database
DATABASE_URL="postgresql://..."

# Firebase (Client)
NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=

# Firebase (Admin SDK)
FIREBASE_PROJECT_ID=
FIREBASE_CLIENT_EMAIL=
FIREBASE_PRIVATE_KEY=

# Cloudinary
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=
CLOUDINARY_API_KEY=
CLOUDINARY_API_SECRET=

# Email (Gmail SMTP)
EMAIL_USER=your@gmail.com
EMAIL_PASS=your-app-password

# Auth
JWT_SECRET=your_32_char_secret

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### 3. Database Setup

```bash
npx prisma db push       # Push schema to your database
npx prisma generate      # Generate Prisma client
```

### 4. Run Locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) — you're live! 🎉

## 🔄 User Flows

### Guest → Customer

```
Landing Page
    ↓ Search "plumber" or click category
/hire?search=plumber  (no login needed)
    ↓ Click a provider
/provider/[id]  (view profile, reviews, pricing — no login)
    ↓ Tap "Book Now"
LoginPromptModal → /login?from=/provider/[id]
    ↓ Sign in
Back to provider → booking modal → booked! ✅
```

### Provider Registration

```
/register → choose "Register as Professional"
    → Set name, category, location, pricing
    → Upload ID Proof + Business License
    → Admin gets email alert 📧
    → Admin approves/rejects in panel
    → Provider gets email with result 📧
    → ✓ Verified badge appears on profile
```

## 📧 Email Notifications

Automated HTML emails are sent at key moments:

| Event | Recipient | Template |
|---|---|---|
| Provider uploads documents | **Admin** | "New Document Awaiting Review" + link |
| Admin approves a document | **Provider** | "Your ID Proof has been approved ✅" |
| Admin approves both docs | **Provider** | "You're now Verified 🏆" |
| Admin rejects a document | **Provider** | "Not approved ❌ — Reason + re-upload link" |

## 📊 Admin Dashboard

The admin panel at `/admin` includes:

- **Stats** — Total users, providers, bookings, pending approvals
- **Analytics** — Bookings trend (7/30/90 day), Revenue by category, Top providers leaderboard, Booking funnel
- **Document Review** — View uploaded ID & license docs, approve or reject with reason
- **Provider Management** — Approve, revoke, verify, remove providers
